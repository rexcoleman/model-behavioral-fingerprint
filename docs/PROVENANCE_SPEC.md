# Provenance Spec — FP-13 Model Behavioral Fingerprinting

## Purpose

Every experiment run must produce provenance artifacts sufficient to reproduce
the exact software environment and codebase state. This closes LL-25
(reproducibility) and LL-27 (environment drift).

## Provenance Directory

All provenance artifacts live in `outputs/provenance/`.

## Required Artifacts

### 1. `versions.txt`
- Generated by: `pip freeze` inside the base conda environment
- Contains: exact package versions for every Python dependency
- Regenerate: `pip freeze > outputs/provenance/versions.txt`

### 2. `git_info.txt`
- Contains: commit SHA, branch name, dirty status
- Format:
  ```
  commit: <full SHA>
  branch: <branch name>
  dirty: <true|false>
  ```
- Regenerate: run `git rev-parse HEAD`, `git branch --show-current`, `git status --porcelain`

### 3. `config_resolved.yaml`
- Contains: resolved experiment configuration (detectors, representations,
  seeds, synthetic data parameters, contrastive learning hyperparameters)
- Source of truth: `project.yaml` + script defaults
- Must be updated when experiment parameters change

## Verification Protocol

Before any FINDINGS.md update or publication:

1. **Environment match:** `pip freeze` output must match `versions.txt` exactly
   (or diff must be reviewed and justified).
2. **Code match:** `git_info.txt` commit must match the commit that produced
   the results in `outputs/`. If dirty=true, the uncommitted changes must be
   documented.
3. **Config match:** `config_resolved.yaml` must match the parameters used in
   the most recent experiment run. Cross-check against JSON result files in
   `outputs/detection/`.

## Staleness Rule

Provenance artifacts are stale if:
- Any dependency is upgraded (versions.txt)
- Any commit is made after the recorded SHA (git_info.txt)
- Any experiment parameter changes (config_resolved.yaml)

Stale provenance must be regenerated before the next experiment phase.

## Automation

The `reproduce.sh` script should regenerate provenance as its first step:
```bash
pip freeze > outputs/provenance/versions.txt
git rev-parse HEAD > /tmp/sha && ...
```
